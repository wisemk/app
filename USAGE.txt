소액뱅크 앱 사용법

1. 관리자 페이지에서 콘텐츠 수정
- Render 웹서비스의 /admin 경로로 접속
- 관리자 계정으로 로그인
- 문구, 전화번호, 상세페이지 링크, 상담 링크, 이미지 URL 수정
- 저장 후 앱에서 새로고침 또는 재실행

2. 관리자 화면 버튼 의미
- 다시 불러오기: 현재 서버에 저장된 값을 다시 읽음
- 저장하기: 지금 화면 값을 서버 JSON에 저장
- 배포 기준 반영: 현재 배포된 content/app-content.json 값으로 서버 값을 덮어씀

3. 앱 UI 빠르게 확인
- Expo Go가 아니라 Dev Build 앱 사용
- 상위 폴더의 run-preview.bat 실행
- 휴대폰의 설치된 Dev Build 앱으로 연결
- JS/UI 수정은 이 방식으로 빠르게 확인

4. EAS 빌드 프로필 사용 기준
- development: Dev Client용 개발 빌드
- preview: 설치 테스트용 APK
- production: Play Store 제출용 AAB

5. 활동 기록 확인 개념
- 앱 실행/복귀 시 기기 등록과 앱 오픈이 서버로 기록됨
- 서버는 Postgres 모드면 영구 저장, memory 모드면 서버 재시작 후 유실
- /health 응답에서 activityStoreMode를 보면 현재 저장소 모드를 알 수 있음

6. 운영 점검
- 서버 상태 확인: /health
- 관리자 저장 후 앱에 반영 안 되면:
  - 앱 새로고침
  - 앱 완전 종료 후 재실행
  - 필요 시 관리자에서 배포 기준 반영

7. Postgres 관련
- 새 DB 생성 후 외부 접속 문자열로 npm run db:init 먼저 실행
- 그 다음 웹서비스 환경변수에 내부 접속 문자열 저장
- /health에서 postgres 모드 확인 후 사용

8. 자주 생기는 문제
- /health가 memory로 뜸:
  - DATABASE_URL 미적용
  - 재배포 미완료
  - 잘못된 환경변수 입력
- 앱에서 예전 문구가 보임:
  - 서버 값이 아직 예전 값
  - 관리자 저장 안 됨
  - 앱이 예전 번들 기준으로 열림
- Dev Build가 안 붙음:
  - run-preview.bat 다시 실행
  - 새 QR 사용
  - Expo Go로 열지 말 것

9. 보안 / 운영 주의
- 관리자 비밀번호와 DB 접속 문자열은 채팅/문서에 직접 붙여넣지 말 것
- 비밀번호나 DB URL이 노출되면 새 credential 발급 후 교체
- 운영 비밀번호는 주기적으로 변경 권장

10. 지금 단계에서 해야 할 우선순위
- DB credential 재발급
- 실제 푸시 발송 기능 추가
- Expo push receipt 확인 로직 추가
- 관리자에서 푸시 캠페인 발송까지 연결
